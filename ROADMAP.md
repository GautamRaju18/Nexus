# Roadmap — how each agent deepens

v1 ships **all 21 agents as real, reasoning specialists** sharing one runtime, plus
a small set of working, no-key tools (web search/fetch, memory, notes). Each agent
becomes *deeper* by granting it more tools in `src/agents/specs.ts` — **no framework
changes**. This is the path from "thin but real" to "full staff".

## How to add a capability (the only pattern you need)
1. Write a tool in `src/core/tools.ts` (or a new file) — id, `sensitivity`, zod
   `input`, `scopes`, `handler`. Mark it `internal: true` only if it touches Nexus's
   own store.
2. Add the tool id to the relevant agent(s) in `src/agents/specs.ts`.
3. Done. The autonomy gate, approvals, and audit log apply automatically.

Store OAuth tokens via `vault.set(...)` so they're encrypted at rest.

## Phase 1 — the personal core (highest daily value)
| Agent | Tools to add | Notes |
|---|---|---|
| Calendar | `gcal_list`, `gcal_create`, `gcal_move` | Google Calendar API (free, your OAuth) |
| Email | `gmail_list`, `gmail_read`, `gmail_draft`, `gmail_send` | `gmail_send` = `comms` (gated) |
| Secretary | `reminder_set` + cron | proactive nudges |
| Finance | `bank_read` (CSV import first, then aggregation) | read-only by default |

## Phase 2 — proactivity & surfaces
- **Event bus + scheduler** (`node-cron` locally, or GitHub Actions when cloud):
  renewal watches, cold-thread nudges, calendar-overload detection, price watches.
- **Telegram surface** (`src/surfaces/telegram.ts`): same orchestrator, message in /
  brief out, approvals as inline buttons.
- **Morning/evening brief** generated by the Chief of Staff.

## Phase 3 — knowledge & production
| Agent | Tools |
|---|---|
| Research | richer search (Apify/SerpAPI free tiers), source verification, `watch_topic` |
| Document | `make_docx`, `make_pptx`, `make_xlsx`, `make_pdf` (generators) |
| Data | `sheet_read`, `sql_query`, charting |

## Phase 4 — business workforce
| Agent | Tools |
|---|---|
| PR / Marketing / Social | contact lists, drafting at scale (batch + cadence), platform post (gated) |
| Recruitment | ATS/job-board read, scorecards, interview scheduling |
| Support / Operations | helpdesk read/reply, KB retrieval, process runners |
| Negotiation / Legal | comparables research, contract red-flagging (advice-only) |

## Phase 5 — autonomy & marketplace
- Per-domain trust meters; propose autonomy increases after clean track records.
- Browser-automation fallback for no-API sites; AI voice for phone-only providers.
- Agent SDK + local "agent store" so new specialists install as scoped specs.

## Cost discipline (staying ₹0)
- Default to the **local Ollama** brain and **local SQLite** — no metered services.
- Act on **your own accounts** (Gmail, Calendar) via OAuth — no per-action vendor fees.
- When a feature would require a paid tier (premium search volume, phone calls,
  always-on cloud), the code should **log the cost line before crossing it** so ₹0 is
  never crossed silently.
